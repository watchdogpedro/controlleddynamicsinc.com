Lessons Learned

What Worked Well

  1. Systematic Diagnosis: Testing tiny commits first quickly identified size as root cause
  2. Incremental Approach: Cherry-picking commits in small batches succeeded where bulk push failed
  3. Documentation First: Writing comprehensive rules prevents future time waste

  5. Metadata Strategy: Specific company names (Boeing, SpaceX) in metadata directly targets high-value customers

Challenges Encountered

  1. Network Limitations Not Immediately Obvious
    • Challenge: Initial assumption was GitHub issue, not local ISP throttling
    • How Overcome: Tested multiple protocols (SSH/HTTPS), multiple approaches (buffer settings, timeouts), compared to Heroku (worked fine) ‚Üí revealed ISP-specific throttling
    • Lesson: Always test alternative services (Heroku) to isolate whether issue is service-specific or network-wide
  2. Large Changesets Accumulated
    • Challenge: 25+ commits accumulated before discovering push issue
    • How Overcome: Batched commits using cherry-pick strategy
    • Lesson: Push more frequently (3-5 commits) to avoid accumulating large changesets
  3. Heroku Deployment Timeouts
    • Challenge: Same network issue affects Heroku deployment
    • How Overcome: Accepted that Heroku may require manual dashboard deployment
    • Lesson: GitHub is source of truth; Heroku can be deployed manually when needed
